# Инструкция для администратора (МойСклад)

## 1. Пользовательские поля
Создайте пользовательские поля с точными названиями:

### Контрагент
- `LoyaltyEnabled` (тип: Логическое)
- `LoyaltyDiscountPercent` (тип: Число)

### Товар
- `IsPromo` (тип: Логическое)
  - Если вместо поля используются теги, укажите название тега в `.env` как `PROMO_TAG`.

### Документ (Заказ покупателя / Отгрузка)
- `DisableLoyalty` (тип: Логическое)
- `LoyaltyDiscountSum` (тип: Число)

## 2. Права доступа
Пользователь, чей токен используется в сервисе, должен иметь права:
- Просмотр и изменение заказов/отгрузок
- Доступ к пользовательским полям
- Доступ к позициям документа

## 3. Вебхуки
Создайте вебхуки на события:
- Создание документа
- Изменение документа

URL вебхука:
```
http://<ваш_хост>:8080/webhook
```

## 4. Проверка
1. Создайте контрагента с включенной программой лояльности.
2. Создайте товар обычный и товар с признаком акции.
3. Создайте документ и добавьте оба товара.
4. Убедитесь, что:
   - обычный товар получил скидку
   - акционный товар не получил скидку
   - в документе заполнено поле `LoyaltyDiscountSum`

## 5. Ручное отключение
Если нужно отключить автоприменение скидки для документа:
- Установите `DisableLoyalty = true`.
- Доступ к изменению поля — только у администратора/руководителя.
